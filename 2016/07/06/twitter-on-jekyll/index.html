如何在 jekyll 中嵌入 twitter

07/06/16 on Technique

怎样在自己的博客中嵌入 twitter 消息呢？

谷歌百度了很多相关资料，没发现相关的中文资料，大概是大家都没这样的需求吧。英文资料倒是很多，但是初步试验都不奏效。最终找了一个 ruby 插件，跟原作者讨论了很久，解决了这个问题，顺带理解了很多东西，在这里记录一下。

第一步：下载插件

如果本地 repo 没有 _plugins 文件夹的话，新建一个

$ mkdir _plugins

然后在 _plugins 下

$ wget https://raw.githubusercontent.com/takuti/jekyll-lazy-tweet-embedding/master/lazy_tweet_embedding.rb

这样一来这个插件就算是安装好了，不过还不能使用。

第二步：调整代理

按照作者的意思，直接在文章中插入推特的 url，然后这个 ruby 插件就可以借用推特官方提供的 API 自动把它转换成相应的可以在网页中显示的推特信息。

然而当你尝试

$ jekyll build

或者 jekyll serve 等等 来创建本地站点的时候，jekyll 会报错提示你 connect timeout(2)，这是因为我朝威武雄壮的万里长城给你把 twitter 挡住了，所以在 lazy_tweet_embedding.rb 文件中

 # convert tweet url to embedding html
  class LazyTweetEmbedding
    def get_html(id)
        url = "https://api.twitter.com/1/statuses/oembed.json?id=#{id}"
        JSON.parse(open(url).read, { :symbolize_names => true })[:html]
    end

    def convert(line)
      r = /^https?:\/\/twitter\.com\/[a-zA-Z0-9_]+\/status\/([0-9]+)\/?$/
      r =~ line ? get_html($~[1]) : line
    end

    def embed(content)
      content.lines.collect {|line| convert(line) }.join
    end
  end

open(url).read 这块没法进行。

好在 open-url 这个函数提供了 proxy 参数可以让我们使用，我们可以把函数改成

JSON.parse(open(url, :proxy => "http://127.0.0.1:1080").read, { :symbolize_names => true })[:html]

127.0.0.1:1080是 shadowsocks 的默认端口。

但是这样改动之后依然报错，connect timeout 变成了 read timeout。

第三步：polipo 对 shadowsocks 进行转换

127.0.0.1:1080 一直是我用 shadowsocks 时填的地址，不知道哪里有问题。到处查资料怎没查出个所以然来。但是看到别人引用shadowsocks 的时候都是写的：socks5://127.0.0.1:1080，我这里函数只能写 http 因为其他的开头无法识别，我才开始意识到，socks5和 http 是两种代理。

由于我这边只有 shadowsocks，所以我只能想办法把 socks 转成 http，我选择采用 polipo。

具体操作可以 戳我，这里讲得非常清楚。

开启 polipo 的时候

$ polipo socksParentProxy=localhost:1080
Established listening socket on port 8123.

把原来的代理导到了 8123端口，所以原来的函数再改一下

JSON.parse(open(url, :proxy => "http://127.0.0.1:8123").read, { :symbolize_names => true })[:html]

每次在 jekyll build 之前先把 polipo 打开。试验一下，到这里，我们已经可以在本地完成工作了。

第四步：真正地静态部署 jekyll

我一开始没有理解这个“静态”的含义。github pages + jekyll 一直说自己是静态的，然后我又是完全按照他们的步骤来的，所以我以为我的站点是静态的了。

事实上，github pages + jekyll 是动态根据你的源文件生成的站点，官方甚至不需要你把 _site 文件夹推上 repo。这样一来，很多插件在 github pages 上就没有办法正常工作了。

想要完全地使用 jekyll 的插件不受 github pages 的限制，但是又想要借用 github pages 的服务器？方法很简单，只把 _site 的内容推上 repo 就可以了。

注意不是 _site 文件夹，是里面的全部内容。
